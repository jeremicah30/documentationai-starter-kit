## Overview

Jeremicah Licup supports seamless integrations with popular tools to streamline your documentation processes. Sync changes from GitHub repositories, send notifications to Slack channels, set up webhooks for real-time updates, embed external content, and leverage the API for custom solutions. These integrations help automate workflows and keep your docs always up-to-date.

<Callout kind="tip">
Start with GitHub integration for automatic version syncing—it's the most common setup.
</Callout>

## GitHub Integration

Connect your GitHub repository to automatically sync documentation updates.

<Steps>
  <Step title="Create GitHub App" icon="settings">
    In your GitHub account, navigate to Settings > Developer settings > GitHub Apps. Create a new app with permissions for repository contents and metadata.
  </Step>
  <Step title="Generate Token" icon="key">
    Generate an installation access token with `repo` scope.
  </Step>
  <Step title="Configure in Jeremicah Licup" icon="link">
    Go to your Jeremicah Licup dashboard at `https://dashboard.example.com/integrations/github`. Paste the token and select your repository.
  </Step>
  <Step title="Test Sync" icon="play">
    Push a commit to your repo and verify the docs update automatically.
  </Step>
</Steps>

<CodeGroup tabs="Node.js,Python">
  ```javascript
  // Example: Webhook handler for GitHub push events
  app.post('/webhook/github', (req, res) => {
    const event = req.headers['x-github-event'];
    if (event === 'push') {
      // Trigger doc rebuild
      rebuildDocs(req.body.repository.full_name);
    }
    res.status(200).send('OK');
  });
  ```
  ```python
  # Example: Webhook handler for GitHub push events
  from flask import Flask, request

  app = Flask(__name__)

  @app.route('/webhook/github', methods=['POST'])
  def github_webhook():
      event = request.headers.get('X-GitHub-Event')
      if event == 'push':
          # Trigger doc rebuild
          rebuild_docs(request.json['repository']['full_name'])
      return 'OK', 200
  ```
</CodeGroup>

## Slack Notifications

Set up Slack alerts for documentation events like publishes or errors.

<Tabs>
  <Tab title="Incoming Webhook" icon="plug">
    Create a Slack app and get your webhook URL from `https://api.slack.com/messaging/webhooks`.

    ```bash
    curl -X POST https://hooks.slack.com/services/YOUR/SLACK/WEBHOOK \
      -H 'Content-type: application/json' \
      --data '{
        "text": "Docs published successfully!"
      }'
    ```

    Add the URL to Jeremicah Licup settings under Integrations > Slack.
  </Tab>
  <Tab title="Bot User" icon="bot">
    Use Slack Bolt for richer interactions.

    <CodeGroup tabs="JavaScript,Python">
      ```javascript
      const { App } = require('@slack/bolt');
      const app = new App({
        token: process.env.SLACK_BOT_TOKEN,
        signingSecret: process.env.SLACK_SIGNING_SECRET
      });

      app.event('app_mention', async ({ event, say }) => {
        await say('Docs update complete!');
      });
      ```
      ```python
      from slack_bolt import App

      app = App(token="xoxb-your-bot-token", signing_secret="your-signing-secret")

      @app.event("app_mention")
      async def handle_app_mention(body, say):
          await say("Docs update complete!")
      ```
    </CodeGroup>
  </Tab>
</Tabs>

<Callout kind="alert">
Never expose your Slack webhook URLs in public repositories. Store them securely as environment variables like `SLACK_WEBHOOK_URL`.
</Callout>

## Webhooks

Jeremicah Licup emits webhooks for events like `doc.published` and `doc.updated`. Configure your endpoint to receive payloads.

```javascript
// Sample webhook payload
{
  "event": "doc.published",
  "data": {
    "id": "doc-123",
    "title": "Quickstart Guide",
    "url": "https://docs.example.com/quickstart"
  },
  "timestamp": 1699123456
}
```

Set up your server to verify signatures using the `X-Jeremicah-Signature` header.

<ParamField header="X-Jeremicah-Signature" param-type="string" required="true">
  HMAC SHA-256 signature of the payload using your integration secret.
</ParamField>

<ParamField body="event" param-type="string" required="true">
  The event type, such as `doc.published`.
</ParamField>

## API Access for Advanced Integrations

Access the full API at `https://api.example.com/v1` for programmatic control.

<Request tabs="cURL,JavaScript">
  ```bash
  curl -X POST https://api.example.com/v1/docs \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{
      "title": "New Doc",
      "content": "Hello from API!"
    }'
  ```
  ```javascript
  const response = await fetch('https://api.example.com/v1/docs', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${YOUR_API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      title: 'New Doc',
      content: 'Hello from API!'
    })
  });
  ```
</Request>

<Response tabs="200,400">
  ```json
  {
    "id": "doc-456",
    "title": "New Doc",
    "status": "published"
  }
  ```
  ```json
  {
    "error": "Invalid API key",
    "code": 401
  }
  ```
</Response>

<ResponseField name="id" field-type="string" required="true">
  Unique document identifier.
</ResponseField>
