## Organizing Documents into Workspaces

Create workspaces to categorize your documentation. Workspaces act as containers for related docs, making navigation intuitive.

<Steps>
  <Step title="Create Workspace" icon="plus">
    Navigate to the dashboard and select "New Workspace".
  </Step>
  <Step title="Add Documents" icon="file-plus">
    Drag and drop files or use the upload button.
  </Step>
  <Step title="Rearrange" icon="move">
    Use drag-and-drop to reorder items within the workspace.
  </Step>
</Steps>

<Callout kind="tip">
  Nest workspaces for hierarchical structures, like projects under teams.
</Callout>

## Version Control for Documents

Every change creates a version snapshot. You review history, compare diffs, and restore previous states.

<CodeGroup tabs="CLI,API">
  ```bash
  # List versions
  jeremicah versions get --doc-id=doc123

  # Restore version
  jeremicah versions restore --doc-id=doc123 --version=v1.2
  ```
  ```javascript
  // Fetch version history
  const versions = await jeremicah.versions.list({ docId: 'doc123' });

  // Restore a version
  await jeremicah.versions.restore({ docId: 'doc123', version: 'v1.2' });
  ```
</CodeGroup>

## Customizing Themes and Layouts

Personalize your docs with custom themes. Adjust colors, fonts, and layouts via a simple config file.

<Expandable title="Advanced Theme Config" default-open="false">
  Create a `{theme.json}` file in your workspace root:

  ````json
  {
    "primaryColor": "#3B82F6",
    "fontFamily": "Inter, sans-serif",
    "layout": {
      "sidebar": true,
      "darkMode": "auto"
    }
  }
  ````

  Apply changes by saving and refreshing.
</Expandable>

## Search and Filtering Tools

Use advanced search to query across workspaces. Filter by tags, dates, authors, or content keywords.

| Filter Type | Example Query | Description |
|-------------|---------------|-------------|
| Keyword    | `API docs`   | Full-text search |
| Tag        | `#urgent`    | Tag-based filtering |
| Date       | `after:2024-01-01` | Time range |
| Author     | `author:alice` | User-specific |

<Callout kind="info">
  Combine filters like `API docs #urgent author:alice` for precise results.
</Callout>
