## Migration Guide

For breaking changes in recent releases, follow these structured steps to update your integration smoothly.

<Steps>
  <Step title="Review Changes" icon="book-open">

  Examine release notes above. Note any deprecated parameters like old token formats.

  </Step>
  <Step title="Update Configuration" icon="settings">

  ```javascript
  // Old way
  const config = { token: 'legacy-token' };

  // New way
  const config = { apiKey: 'YOUR_API_KEY' };
  ```

  Replace legacy tokens with new `{API_KEY}` values.

  </Step>
  <Step title="Test Integration" icon="play-circle">

  Run your test suite against `https://api.example.com/v1/test-endpoint`.

  </Step>
  <Step title="Deploy" icon="rocket">

  Deploy to production and monitor logs for `{30 minutes}`.

  </Step>
</Steps>

<Expandable title="Advanced Migration for Webhooks" default-open="false">

If you use webhooks, update your endpoint to handle new payload structure:

````javascript
// New webhook payload
{
  "event": "doc.updated",
  "data": {
    "id": "doc-123",
    "changes": ["title", "content"]
  }
}
````

Configure at `https://dashboard.example.com/webhooks`.

</Expandable>
